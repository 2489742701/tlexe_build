# 多选与撤回功能修复记录

## 修复日期
2026-03-21

## 问题描述

用户报告了两个主要问题：
1. **撤回功能问题**：撤回/重做功能无法正常工作
2. **多选拖动问题**：多选组件后无法拖动，虽然可以多选

## 问题分析

### 1. 撤回功能问题

**根本原因**：`project_controller.py` 中的 `_on_undo()` 和 `_on_redo()` 方法只是调用了 `undo_manager.undo()` 和 `undo_manager.redo()`，但没有正确处理返回值。

**原代码问题**：
```python
def _on_undo(self):
    """撤销。"""
    self._undo_manager.undo()  # 只调用，没有检查返回值
```

### 2. 多选拖动问题

**根本原因**：`canvas.py` 中的 `ComponentGraphicsItem.mousePressEvent()` 在点击组件时会发射 `selected` 信号，这会触发 `select_item()` 方法清除其他所有选中项。

**原代码问题**：
```python
def mousePressEvent(self, event):
    ...
    else:
        super().mousePressEvent(event)
        self.selected.emit(self.component_id)  # 这会清除其他选中项
```

## 修复方案

### 1. 撤回功能修复

**文件**：`controllers/project_controller.py`

**修改内容**：

1. 修改 `_on_undo()` 和 `_on_redo()` 方法，检查返回值：
```python
def _on_undo(self):
    """撤销。"""
    if self._undo_manager.undo():
        desc = self._undo_manager.get_undo_description()
        self.window.show_status_message(f"撤销: {desc}" if desc else "撤销")
    else:
        self.window.show_status_message("无法撤销")
```

2. 修改 `_on_add_component()` 使用正确的 `push` 方法签名：
```python
def _on_add_component(self, comp_type: str, parent_id: str):
    """添加组件。"""
    comp = create_component(comp_type)
    comp_data = comp.to_dict()
    self.project_model.add_component(comp)
    
    def undo_add(data):
        comp_id = data.get('id')
        if comp_id:
            self.project_model.remove_component(comp_id)
    
    def redo_add(data):
        new_comp = ComponentModel.from_dict(data)
        self.project_model.add_component(new_comp)
    
    self._undo_manager.push(
        action_type='add',
        description=f"添加 {comp.name}",
        undo_data=comp_data,
        redo_data=comp_data,
        undo_callback=undo_add,
        redo_callback=redo_add
    )
```

3. 修改 `_on_delete_component()` 同样使用回调函数方式：
```python
def _on_delete_component(self, comp_id: str):
    """删除组件。"""
    comp = self.project_model.get_component(comp_id)
    if comp:
        comp_data = comp.to_dict()
        if self.project_model.remove_component(comp_id):
            
            def undo_delete(data):
                new_comp = ComponentModel.from_dict(data)
                self.project_model.add_component(new_comp)
            
            def redo_delete(data):
                cid = data.get('id')
                if cid:
                    self.project_model.remove_component(cid)
            
            self._undo_manager.push(
                action_type='delete',
                description=f"删除 {comp.name}",
                undo_data=comp_data,
                redo_data=comp_data,
                undo_callback=undo_delete,
                redo_callback=redo_delete
            )
```

### 2. 多选拖动修复

**文件**：`views/canvas.py`

**修改内容**：

1. 新增信号和实例变量：
```python
class ComponentGraphicsItem(QGraphicsObject):
    ...
    multi_move_finished = pyqtSignal(dict)  # 新增信号
    
    def __init__(self, ...):
        ...
        # 新增变量
        self._dragging = False
        self._drag_start_pos = QPointF()
        self._selected_items_start_pos = {}
```

2. 修改 `mousePressEvent()` 逻辑：
```python
def mousePressEvent(self, event):
    ...
    modifiers = event.modifiers()
    is_multi_select = (modifiers & Qt.KeyboardModifier.ControlModifier) or \
                     (modifiers & Qt.KeyboardModifier.ShiftModifier)
    
    if self.scene():
        selected_items = [item for item in self.scene().selectedItems() 
                         if isinstance(item, ComponentGraphicsItem)]
        
        if is_multi_select:
            # 多选模式：切换当前项的选中状态
            self.setSelected(not self.isSelected())
        else:
            # 普通选择：只选中当前项，除非当前项已经在选中列表中
            if self not in selected_items:
                self.selected.emit(self.component_id)
        
        # 启动拖动
        selected_items = [item for item in self.scene().selectedItems() 
                         if isinstance(item, ComponentGraphicsItem)]
        
        if len(selected_items) > 0:
            self._dragging = True
            self._drag_start_pos = event.scenePos()
            self._selected_items_start_pos = {
                item.component_id: (item.pos().x(), item.pos().y())
                for item in selected_items
            }
```

3. 修改 `mouseMoveEvent()` 支持多选拖动：
```python
def mouseMoveEvent(self, event):
    if self._resizing:
        self._do_resize(event.scenePos())
    elif self._dragging and self.scene():
        delta = event.scenePos() - self._drag_start_pos
        
        selected_items = [item for item in self.scene().selectedItems() 
                         if isinstance(item, ComponentGraphicsItem)]
        
        for item in selected_items:
            if item.component_id in self._selected_items_start_pos:
                start_x, start_y = self._selected_items_start_pos[item.component_id]
                new_x = start_x + delta.x()
                new_y = start_y + delta.y()
                item.setPos(QPointF(new_x, new_y))
        
        event.accept()
    else:
        super().mouseMoveEvent(event)
```

4. 修改 `mouseReleaseEvent()` 发射多选移动结束信号：
```python
def mouseReleaseEvent(self, event):
    ...
    elif self._dragging:
        self._dragging = False
        
        if self.scene():
            selected_items = [item for item in self.scene().selectedItems() 
                             if isinstance(item, ComponentGraphicsItem)]
            
            is_multi_move = len(selected_items) > 1
            
            new_positions = {}
            for item in selected_items:
                new_pos = item.pos()
                new_positions[item.component_id] = (int(new_pos.x()), int(new_pos.y()))
                item._check_and_update_parent()
                
                # 单选移动时发射 moved 信号，多选移动时不发射
                if not is_multi_move:
                    item.moved.emit(item.component_id, int(new_pos.x()), int(new_pos.y()))
            
            # 多选移动时发射 multi_move_finished 信号
            if is_multi_move and self._selected_items_start_pos:
                self.multi_move_finished.emit({
                    'old': self._selected_items_start_pos,
                    'new': new_positions
                })
```

### 3. 撤回记录支持

**文件**：`controllers/project_controller.py`

**修改内容**：

1. 在 `_on_component_added()` 中连接信号：
```python
def _on_component_added(self, comp_id: str):
    """组件添加后的回调。"""
    comp = self.project_model.get_component(comp_id)
    if comp:
        item = self.window.designer_view.add_component_item(comp)
        if item:
            item.moved.connect(self._on_item_moved)  # 单选移动信号
            item.multi_move_finished.connect(self._on_multi_move_finished)  # 多选移动信号
```

2. 新增单选移动回调方法：
```python
def _on_item_moved(self, comp_id: str, new_x: int, new_y: int):
    """单个组件移动后的回调。"""
    comp = self.project_model.get_component(comp_id)
    if comp:
        old_x, old_y = comp.x, comp.y
        if old_x != new_x or old_y != new_y:
            comp.set_position(new_x, new_y)
            
            def undo_move(data):
                c = self.project_model.get_component(data.get('id'))
                if c:
                    c.set_position(data.get('old_x', 0), data.get('old_y', 0))
                    item = self.window.designer_view.get_item(data.get('id'))
                    if item:
                        item.setPos(data.get('old_x', 0), data.get('old_y', 0))
            
            def redo_move(data):
                c = self.project_model.get_component(data.get('id'))
                if c:
                    c.set_position(data.get('new_x', 0), data.get('new_y', 0))
                    item = self.window.designer_view.get_item(data.get('id'))
                    if item:
                        item.setPos(data.get('new_x', 0), data.get('new_y', 0))
            
            self._undo_manager.push(
                action_type='move',
                description=f"移动 {comp.name}",
                undo_data={'id': comp_id, 'old_x': old_x, 'old_y': old_y, 'new_x': new_x, 'new_y': new_y},
                redo_data={'id': comp_id, 'old_x': old_x, 'old_y': old_y, 'new_x': new_x, 'new_y': new_y},
                undo_callback=undo_move,
                redo_callback=redo_move
            )
```

3. 新增多选移动回调方法：
```python
def _on_multi_move_finished(self, positions: dict):
    """多选移动完成后的回调。"""
    old_positions = positions.get('old', {})
    new_positions = positions.get('new', {})
    
    if old_positions and new_positions:
        
        def undo_move(data):
            self._restore_component_positions(data.get('old', {}))
        
        def redo_move(data):
            self._restore_component_positions(data.get('new', {}))
        
        self._undo_manager.push(
            action_type='move',
            description=f"移动 {len(new_positions)} 个组件",
            undo_data={'old': old_positions, 'new': new_positions},
            redo_data={'old': old_positions, 'new': new_positions},
            undo_callback=undo_move,
            redo_callback=redo_move
        )
```

4. 新增辅助方法 `_restore_component_positions()`：
```python
def _restore_component_positions(self, positions: dict):
    """恢复组件位置。"""
    for comp_id, pos_data in positions.items():
        comp = self.project_model.get_component(comp_id)
        if comp:
            x, y = pos_data if isinstance(pos_data, (list, tuple)) else (pos_data.get('x', 0), pos_data.get('y', 0))
            comp.set_position(int(x), int(y))
            item = self.window.designer_view.get_item(comp_id)
            if item:
                item.setPos(int(x), int(y))
```

## UndoManager 接口说明

`UndoManager` 使用回调函数方式：

```python
def push(self, action_type: str, description: str, 
         undo_data: Dict[str, Any], redo_data: Dict[str, Any],
         undo_callback: Callable, redo_callback: Callable):
    """推送一个操作到撤销栈。"""
```

## 使用说明

### 多选操作
1. 按住 **Ctrl** 键点击组件可以多选/取消选中
2. 按住 **Shift** 键点击组件可以多选/取消选中
3. 使用鼠标框选（橡皮筋选择）可以多选
4. 多选后可以直接拖动所有选中组件

### 撤回/重做
- **Ctrl+Z**：撤销上一步操作
- **Ctrl+Y** 或 **Ctrl+Shift+Z**：重做
- 支持撤销的操作：
  - 组件移动（包括多选移动）
  - 添加组件
  - 删除组件

## 注意事项

1. 撤回管理器最大保存步数为50步（可在 `UndoManager` 构造函数中修改）
2. 多选移动时，只有当实际发生位置变化时才会记录到撤回历史
3. 每次新操作会清空重做栈
4. 多选移动时不会为每个组件单独发射 `moved` 信号，而是统一发射 `multi_move_finished` 信号
