## 一、技术类控件

### 1.1 概念

技术类控件是一体整合控件，点击后可以自动生成多个相关联的控件。它们具有以下特点：

- **一体整合**：多个控件组合成一个整体
- **自动布局**：自动设置控件的位置和大小
- **预设联动**：控件之间有预设的联动关系
- **快速开发**：一键生成复杂功能模块

### 1.2 可用模板

#### 1.2.1 抽奖系统 (lottery)

**描述**：包含标题、轮播、按钮、结果标签的完整抽奖功能

**组件列表**：
- 抽奖标题 (Label)
- 候选人轮播 (ImageCarousel)
- 开始抽奖按钮 (Button)
- 抽奖结果标签 (Label)

**布局**：300x320像素

**使用场景**：年会抽奖、随机选择等

#### 1.2.2 登录表单 (login)

**描述**：包含用户名、密码输入框和登录按钮的完整登录功能

**组件列表**：
- 登录标题 (Label)
- 用户名标签 (Label)
- 用户名输入框 (Input)
- 密码标签 (Label)
- 密码输入框 (Input)
- 登录按钮 (Button)

**布局**：290x190像素

**使用场景**：用户登录、身份验证等

#### 1.2.3 表单组 (form)

**描述**：包含标签和输入框的标准表单行

**组件列表**：
- 字段标签 (Label)
- 字段输入框 (Input)

**布局**：310x30像素

**使用场景**：表单字段、数据输入等

#### 1.2.4 进度面板 (progress_panel)

**描述**：包含标题、进度条和状态标签的进度显示面板

**组件列表**：
- 进度标题 (Label)
- 进度条 (ProgressBar)
- 状态标签 (Label)

**布局**：300x80像素

**使用场景**：任务进度、加载状态等

### 1.3 使用方法

#### 在组件面板中使用

1. 打开组件面板
2. 找到"技术类"分类
3. 点击需要的技术类控件
4. 系统自动生成所有相关组件

#### 在代码中使用

```python
from models import TechComponentManager

# 获取所有可用模板
templates = TechComponentManager.get_available_templates()
print(templates)  # ['lottery', 'login', 'form', 'progress_panel']

# 创建抽奖系统组件
components = TechComponentManager.create_tech_component(
    template_id="lottery",
    parent_id="container_001",
    offset_x=50,
    offset_y=50
)

# components 是一个包含4个组件的列表
for comp in components:
    print(f"{comp.name} ({comp.type}): {comp.width}x{comp.height}")
```


## 二、组节点功能

### 2.1 概念

组节点是一个特殊的组件类型，用于将多个组件组合在一起。它提供以下功能：

- **组件分组**：将相关组件组织在一起
- **布局管理**：支持垂直、水平、网格等布局模式
- **父子关系**：管理组件的父子层级
- **统一操作**：对整组组件进行移动、删除等操作

### 2.2 组节点属性

| 属性 | 类型 | 说明 |
|------|------|------|
| children | List[str] | 子组件ID列表 |
| layout_mode | str | 布局模式 (none, vertical, horizontal, grid) |
| spacing | int | 子组件间距 |
| padding | int | 内边距 |
| auto_size | bool | 是否自动调整大小 |
| show_border | bool | 是否显示边框 |
| border_style | str | 边框样式 (solid, dashed, dotted, none) |

### 2.3 使用方法

#### 在组件面板中使用

1. 打开组件面板
2. 找到"容器类"分类
3. 点击"组节点"
4. 在属性面板中设置布局模式和样式

#### 在代码中使用

```python
from models import GroupNodeModel, create_component

# 创建组节点
group = GroupNodeModel(
    name="表单组",
    x=10,
    y=10,
    width=400,
    height=300
)

# 设置布局模式
group.layout_mode = "vertical"
group.spacing = 15
group.padding = 20

# 添加子组件
button1 = create_component("button", name="按钮1")
button2 = create_component("button", name="按钮2")
label1 = create_component("label", name="标签1")

group.add_child(button1.id)
group.add_child(button2.id)
group.add_child(label1.id)

# 检查子组件
print(f"子组件数量: {len(group.children)}")
print(f"是否包含按钮1: {group.has_child(button1.id)}")

# 移除子组件
group.remove_child(button2.id)
```

### 2.4 布局模式

#### 无布局 (none)

子组件自由定位，不自动排列。

#### 垂直布局 (vertical)

子组件按垂直方向排列，从上到下。

```
[组件1]
[组件2]
[组件3]
```

#### 水平布局 (horizontal)

子组件按水平方向排列，从左到右。

```
[组件1] [组件2] [组件3]
```

#### 网格布局 (grid)

子组件按网格排列。

```
[组件1] [组件2]
[组件3] [组件4]
```

## 三、组件父子关系

### 3.1 概念

组件可以设置父子关系，实现层级管理。父组件可以是容器或组节点。

### 3.2 设置父子关系

```python
from models import create_component

# 创建父容器
parent = create_component("container", name="父容器")

# 创建子组件并设置父ID
child1 = create_component("button", name="按钮1")
child1.parent_id = parent.id

child2 = create_component("label", name="标签1")
child2.parent_id = parent.id
```

### 3.3 父子关系的作用

- **布局约束**：子组件的位置相对于父组件
- **显示控制**：父组件隐藏时，子组件也会隐藏
- **事件传递**：子组件的事件可以传递给父组件
- **批量操作**：可以同时操作父组件和所有子组件

## 六、注意事项

1. **技术类控件**：点击后会生成多个组件，支持撤销操作
2. **组节点**：删除组节点不会删除子组件，需要手动删除
3. **父子关系**：设置父ID时，确保父组件存在
4. **布局模式**：组节点的布局模式需要手动触发排列
5. **性能**：大量组件时，建议使用组节点进行分组管理

### 8.2 创建表单组

```python
from models import GroupNodeModel, create_component

# 创建组节点
form_group = GroupNodeModel(
    name="用户信息表单",
    x=50,
    y=50,
    width=400,
    height=300
)
form_group.layout_mode = "vertical"
form_group.spacing = 10
form_group.padding = 15

# 创建表单字段
fields = [
    ("姓名：", "请输入姓名"),
    ("年龄：", "请输入年龄"),
    ("邮箱：", "请输入邮箱"),
]

for label_text, placeholder in fields:
    # 创建标签
    label = create_component("label", name=f"{label_text}标签")
    label.text = label_text
    label.alignment = "right"
    
    # 创建输入框
    input_field = create_component("input", name=f"{label_text}输入框")
    input_field.placeholder = placeholder
    
    # 添加到组节点
    form_group.add_child(label.id)
    form_group.add_child(input_field.id)

# 添加到项目
project.add_component(form_group)
```

## 九、常见问题

### Q1: 技术类控件可以修改吗？

A: 可以。生成后，每个组件都可以单独修改。

### Q2: 组节点可以嵌套吗？

A: 可以。组节点可以包含其他组节点。

### Q3: 如何删除组节点及其所有子组件？

A: 需要手动删除组节点和所有子组件。

### Q4: 技术类控件模板可以保存吗？

A: 目前不支持保存，但可以通过代码注册自定义模板。

### Q5: 组节点的布局模式如何生效？

A: 目前需要手动排列子组件，未来版本将支持自动布局。
