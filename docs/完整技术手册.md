## 一、项目概述

**项目名称**: 傻瓜桌面开发工具 (UI快速开发工具)

**项目类型**: PyQt6 桌面应用程序可视化开发工具

**主要功能**:
- 可视化界面设计器
- 组件拖拽、调整大小、属性编辑
- Galgame模式：主程序 → 按钮 → 事件窗口的分支结构
- 逻辑树视图展示程序流程
- 流程图预览
- 项目保存/加载 (.fool 格式)
- 导出为独立Python脚本

### 5.1 数据模型层 (models/)

组件基类，所有UI组件的父类。

**关键属性**:

| `id` | str | 唯一标识符（8位UUID） |
| `type` | str | 组件类型（button/label/input/container） |
| `name` | str | 组件名称 |
| `x, y` | int | 画布坐标 |
| `width, height` | int | 尺寸 |
| `text` | str | 显示文本 |
| `parent_id` | str | 父容器ID（用于层级关系） |
| `style` | StyleConfig | 样式配置 |
| `action` | ActionConfig | 行为配置 |
| `visible` | bool | 可见性 |
| `enabled` | bool | 可用性 |

**关键信号**:
- `data_changed`: 数据变化
- `position_changed(x, y)`: 位置变化
- `size_changed(w, h)`: 尺寸变化

#### ProjectModel (base.py)
项目模型，管理整个工程数据。

**关键属性**:
| 属性 | 类型 | 说明 |
|------|------|------|
| `name` | str | 项目名称 |

| `current_window_id` | str | 当前编辑窗口 |
| `main_window_id` | str | 主程序窗口 |

**关键方法**:
- `add_component(comp, window_id)`: 添加组件

## 六、核心功能实现

### 6.1 组件拖入容器

**实现位置**: `canvas.py` - `ComponentGraphicsItem._check_and_update_parent()`

**工作原理**:
1. 组件拖动结束后，获取组件中心点的场景坐标
2. 遍历该位置下的所有图形项
3. 检测是否在容器内部（排除标题栏区域）
4. 更新组件的 `parent_id` 属性
5. 发射 `parent_changed` 信号

**代码片段**:
```python
def _check_and_update_parent(self):
    if self._model.type == "container":
        return
    
    scene_pos = self.sceneBoundingRect().center()
    
    for item in self.scene().items(scene_pos):
        if isinstance(item, ComponentGraphicsItem) and item != self:
            if item.model.type == "container":
                container_rect = item.sceneBoundingRect()
                title_bar_rect = QRectF(
                    container_rect.left(), container_rect.top(),
                    container_rect.width(), self.TITLE_BAR_HEIGHT
                )
                
                if container_rect.contains(scene_pos) and not title_bar_rect.contains(scene_pos):
                    self._model.parent_id = item.model.id
                    self.parent_changed.emit(self.component_id, item.model.id)
                    return
    
    if self._model.parent_id:
        self._model.parent_id = ""
        self.parent_changed.emit(self.component_id, "")
```

### 6.2 组件层级管理

**实现位置**: `canvas.py` - `DesignerView._update_z_values()`

**工作原理**:
1. 将所有组件分为容器和其他两类
2. 容器设置较低的Z值（从0开始）
3. 其他组件设置较高的Z值（从容器的数量+1开始）

**代码片段**:
```python
def _update_z_values(self):
    containers = []
    others = []
    
    for item in self._items.values():
        if item.model.type == "container":
            containers.append(item)
        else:
            others.append(item)
    
    for i, item in enumerate(containers):
        item.setZValue(i)
    
    base_z = len(containers)
    for i, item in enumerate(others):
        item.setZValue(base_z + i + 1)
```

### 6.3 按钮事件分支

**实现位置**: `models/base.py` - `ProjectModel.create_event_for_button()`

**工作原理**:
1. 创建新的 WindowModel（类型为 EVENT）
2. 设置按钮的 `target_window_id` 和 `branch_name`
3. 将窗口添加到项目
4. 逻辑树自动显示分支关系

### 6.4 项目运行

**实现位置**: `controllers/project_controller.py` - `_on_run_project()`

**工作原理**:
1. 将项目数据序列化为JSON
2. 保存到临时文件（.fool格式）
3. 启动 `runtime/runner.py` 运行时
4. 运行时加载项目并创建实际窗口

### 10.1 文件编码
- 所有Python文件应使用UTF-8编码
- **避免BOM字符**，否则会报语法错误

### 10.2 信号连接
组件添加到画布后需要连接四个信号：
```python
item.moved.connect(self._on_item_moved)
item.selected.connect(self._on_item_selected)
item.resized.connect(self._on_item_resized)
item.parent_changed.connect(self._on_item_parent_changed)
```

### 10.3 层级关系
- 组件的 `parent_id` 用于表示容器关系
- 影响逻辑树显示和组件位置计算

### 10.4 Z值管理
- 每次添加组件后需要调用 `_update_z_values()` 更新层级
- 容器Z值较小，其他组件Z值较大

### 10.5 网格吸附
- 组件移动时自动吸附到网格（默认10px）
- 可通过设置关闭

### 10.6 图形项更新
在 PyQt6 中，移动图形项后需要调用 `scene().update()` 触发重绘，否则会产生残影：
```python
if change == QGraphicsObject.GraphicsItemChange.ItemPositionHasChanged and self.scene():
    self.scene().update()
```

### 10.7 信号阻塞
在批量修改模型属性时使用 `blockSignals(True/False)` 避免重复触发更新：
```python
self._model.blockSignals(True)
try:
    self._model.width = new_width
    self._model.x = new_x
finally:
    self._model.blockSignals(False)
```

## 十一、常见问题

### Q: 组件无法拖入容器？
A: 检查 `_check_and_update_parent()` 方法是否正确调用，确保 `parent_changed` 信号已连接。

### Q: 组件显示在容器下方？
A: 检查 `_update_z_values()` 是否被调用，确保容器Z值较小。

### Q: 拖拽后有残影？
A: 确保 `ItemPositionHasChanged` 时调用了 `scene().update()`。

### Q: 运行项目报错？
A: 检查 `runtime/runner.py` 是否存在，确保项目数据格式正确。

### Q: 文件保存后乱码？
A: 确保使用UTF-8编码保存，避免BOM字符。
