# 窗口编辑器预览与导出尺寸不一致问题分析报告（更新版）

## 问题概述

在傻瓜桌面开发工具中，编辑器画布上设计的界面与运行时预览/导出的实际窗口存在尺寸和位置不一致的问题。具体表现为：
- 组件位置向内收缩（预览画面仿佛编辑器放大了一倍）
- 左上角、左下角创建的组件都会向里缩
- 窗口尺寸与编辑器不匹配

---

## 最新代码分析

### 当前代码逻辑（runner.py）

**窗口尺寸设置**（第89-93行）：
```python
title_bar_height = 28
width = container_data.get('width', 800)
height = container_data.get('height', 600) - title_bar_height  # 减去标题栏高度
self.resize(width, height)
```

**坐标转换**（第141-145行）：
```python
rel_x = comp_x - container_x
rel_y = comp_y - container_y - title_bar_height

comp_data_copy['x'] = max(0, rel_x)
comp_data_copy['y'] = max(0, rel_y)
```

---

## 核心问题分析

### 问题1：窗口高度计算错误

**错误逻辑**：
```
运行时窗口高度 = 编辑器容器高度 - 28
```

**问题说明**：

| 对比项 | 编辑器容器 | 运行时窗口 |
|--------|-----------|-----------|
| 总高度 | 600px（含标题栏28px） | 572px（600-28） |
| 内容区域高度 | 572px（600-28） | 572px |
| 标题栏 | 由容器自己绘制（28px） | 由Qt/系统提供（约30px） |

**结果**：运行时窗口实际可用空间比编辑器显示的**小约30像素**（系统标题栏高度）。

### 问题2：双重标题栏问题

编辑器中的容器：
- 自己绘制标题栏（28px）
- 内容区域从 y=28 开始

运行时窗口：
- Qt/系统提供标题栏（约30px）
- 内容区域从 y=0 开始
- 但代码又减去了28px，导致内容区域进一步缩小

### 问题3：坐标系统不匹配

**编辑器坐标系统**：
```
桌面坐标系（0,0）在左上角
┌────────────────────────────────────┐
│  桌面 (1920×1080)                  │
│     ┌─────────────────┐            │
│     │ 容器 (600×400)  │            │
│     │ ┌─────────────┐ │            │
│     │ │ 标题栏 28px │ │            │
│     │ ├─────────────┤ │            │
│     │ │ 内容区域    │ │            │
│     │ │ 组件在这里  │ │            │
│     │ └─────────────┘ │            │
│     └─────────────────┘            │
└────────────────────────────────────┘
```

**运行时坐标系统**：
```
窗口坐标系（0,0）在内容区域左上角
┌─────────────────┐
│ 系统标题栏      │  ← Qt/系统提供
├─────────────────┤
│ 内容区域        │  ← 组件应该在这里
│ (0,0) 在这里    │
│ 组件在这里      │
└─────────────────┘
```

### 问题4：具体场景分析

**场景：组件在容器左上角**

假设：
- 容器位置: (500, 300)
- 容器尺寸: 400×300
- 组件位置: (510, 340)（紧贴容器内容区域左上角）

当前代码计算：
```
rel_x = 510 - 500 = 10
rel_y = 340 - 300 - 28 = 12
```

**结果**：组件在运行时窗口的 (10, 12) 位置。

**问题**：如果窗口高度被减去28px，而系统标题栏又占用了约30px，实际可用空间变小了。

## 问题验证

### 测试用例：不同位置的组件

| 组件位置 | 编辑器坐标 | 转换后坐标 | 预期问题 |
|----------|-----------|-----------|---------|
| 左上角 | (container_x+10, container_y+38) | (10, 10) | 正常 |
| 右上角 | (container_x+width-60, container_y+38) | (width-60, 10) | 可能超出边界 |
| 左下角 | (container_x+10, container_y+height-40) | (10, height-68) | 可能超出边界 |
| 右下角 | (container_x+width-60, container_y+height-40) | (width-60, height-68) | 可能超出边界 |

**关键发现**：
- 当组件在容器底部时，`rel_y = comp_y - container_y - 28`
- 如果 `comp_y` 接近 `container_y + height`，则 `rel_y` 接近 `height - 28`
- 但窗口高度已经减去了28，所以 `rel_y` 可能超出窗口内容区域

## 根本原因总结

### 原因1：窗口高度不应减去标题栏

**错误代码**：
```python
height = container_data.get('height', 600) - title_bar_height  # ❌ 错误
```

**正确做法**：
```python
height = container_data.get('height', 600)  # ✅ 不减去标题栏
```

**理由**：
- 编辑器容器的标题栏是自绘的，高度包含在容器总高度中
- 运行时窗口的标题栏由系统提供，不占用窗口客户区高度
- 两者应该保持一致的内容区域高度

### 原因2：坐标转换逻辑需要调整

**当前逻辑**：
```python
rel_y = comp_y - container_y - title_bar_height
```

**问题**：
- 这个转换假设编辑器中内容区域从 y=28 开始
- 但如果窗口高度不减去28，这个转换仍然正确
- 问题在于窗口高度和坐标转换不匹配

## 修复方案

### 方案A：保持窗口高度不变（推荐）

**修改 runner.py 第89-93行**：
```python
title = container_data.get('text', '未命名窗口')
self.setWindowTitle(title)

width = container_data.get('width', 800)
height = container_data.get('height', 600)  # 不减去标题栏高度

self.resize(width, height)
```

**说明**：
- 运行时窗口高度 = 编辑器容器高度
- 系统标题栏不占用客户区高度
- 内容区域高度一致

### 方案B：同步调整坐标转换

如果保持窗口高度减去28px，则需要调整组件Y坐标的计算：

**修改 runner.py 第141-145行**：
```python
rel_x = comp_x - container_x
# 如果窗口高度减去了28，这里就不应该再减28
rel_y = comp_y - container_y  # 不减去标题栏高度

comp_data_copy['x'] = max(0, rel_x)
comp_data_copy['y'] = max(0, rel_y)
```

## 推荐修复方案

**同时修改两处**：

1. **窗口高度**：不减去标题栏
2. **坐标转换**：保持减去标题栏（因为编辑器中内容区域确实从y=28开始）

```python
# 窗口尺寸设置
width = container_data.get('width', 800)
height = container_data.get('height', 600)  # 不减去标题栏

# 坐标转换
rel_y = comp_y - container_y - title_bar_height  # 保持减去标题栏
```

**原理**：
- 编辑器容器高度600 = 标题栏28 + 内容区域572
- 运行时窗口高度600 = 系统标题栏（不占客户区） + 内容区域600
- 组件Y坐标从编辑器的绝对坐标转换为相对于内容区域的坐标

## 总结

**核心问题**：窗口高度减去了编辑器容器的标题栏高度（28px），但运行时窗口的标题栏由系统提供，不占用客户区高度，导致内容区域尺寸不匹配。

**表现**：
- 组件位置向内收缩
- 底部组件可能超出窗口边界
- 预览画面与编辑器不一致

**修复**：
1. 窗口高度不减去标题栏高度
2. 坐标转换保持减去标题栏（因为编辑器中内容区域从y=28开始）
