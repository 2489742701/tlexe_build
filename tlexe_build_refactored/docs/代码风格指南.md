# 代码风格指南

本文档定义了项目 `tlexe_build_refactored` 的代码风格规范，所有贡献者应遵循。

---

## 1. 命名规范

### 1.1 类名：PascalCase

```python
# 正确
class ButtonModel(ComponentModel):
class SignalConnectionSchema(BaseModel):
class ActionType(str, Enum):

# 错误
class button_model:
class signal_Connection:
```

### 1.2 函数/方法名：snake_case

```python
# 正确
def set_position(self, x: int, y: int) -> None:
def _sync_item_labels(self) -> None:

# 错误
def SetPosition:
def syncItemLabels:
```

### 1.3 变量名：snake_case

```python
# 正确
comp_data = comp.to_dict()
current_window_id = self._current_window_id

# 错误
compData = comp.toDict()
currentWindowID = self._currentWindowID
```

### 1.4 常量名：UPPER_SNAKE_CASE

```python
# 模块级常量
DEFAULT_DESKTOP_WIDTH = 1280
TECH_TEMPLATES = {}

# 类级常量
class ComponentGraphicsItem:
    MIN_SIZE = 20
    TITLE_BAR_HEIGHT = 28
```

### 1.5 私有成员：单下划线前缀

```python
# 正确 — 私有属性和方法
self._loading = True
self._components: Dict[str, ComponentModel] = {}
def _sync_item_labels(self) -> None:

# SignalProperty 的存储属性也遵循此模式
# SignalProperty(name='x') → 存储 self._x
```

**不要使用双下划线前缀**（Python 名称改写会带来不必要的复杂性）。

### 1.6 Qt Signal 命名：snake_case + 过去分词/名词

```python
# 属性变更信号：名词_changed
data_changed = Signal()
position_changed = Signal(int, int)
style_changed = Signal()

# 集合变更信号：名词_动词过去分词
component_added = Signal(str)
component_removed = Signal(str)

# 用户交互信号：简短动词过去分词
moved = Signal(str, int, int)
resized = Signal(dict)

# 生命周期信号：名词_动词过去分词
project_loaded = Signal()
project_saved = Signal()
```

## 2. 导入风格

### 2.1 三段式导入（PEP 8）

```python
# 1. 标准库
import uuid
import json
from typing import Dict, Any, List, Optional

# 2. 第三方库
from PySide6.QtCore import QObject, Signal
from pydantic import BaseModel, Field

# 3. 本地模块
from .schemas import StyleConfig, ActionConfig
from models import ComponentModel, create_component
```

### 2.2 延迟导入

为避免循环依赖，允许在函数/方法内部使用延迟导入：

```python
def _on_run_project(self):
    from runtime.runner import Runner  # 延迟导入，避免循环依赖
    runner = Runner()
    runner.run(project_data)
```

但应尽量减少使用，优先考虑重构模块结构。

### 2.3 包级重导出

`models/__init__.py` 集中重导出所有公共符号，外部代码使用：

```python
from models import ButtonModel  # 而非 from models.components import ButtonModel
```

## 3. 文档字符串

### 3.1 语言：简体中文

所有文档字符串使用简体中文。

### 3.2 格式：Google Style

```python
def create_connection(self, source_id: str, signal_type: SignalType,
                      target_id: str, action_type: CommActionType,
                      action_params: Optional[Dict[str, Any]] = None) -> str:
    """创建信号连接。

    Args:
        source_id: 源组件ID
        signal_type: 信号类型
        target_id: 目标组件ID
        action_type: 动作类型
        action_params: 动作参数

    Returns:
        创建的连接ID
    """
```

### 3.3 模块文档字符串

每个模块开头应有中文文档字符串，说明核心设计和用途：

```python
"""组件模型模块。

所有组件模型使用 SignalProperty 声明属性，自动获得：
- 属性变更时 emit 信号
- to_dict/from_dict 自动序列化（基类处理）
- 类型注解

子类只需声明 SignalProperty 和业务方法，无需手写 property/setter/to_dict/from_dict。
"""
```

### 3.4 类文档字符串

简短描述类的用途，复杂类可包含设计说明：

```python
class SignalProperty:
    """描述符：属性变更时自动发射 Qt 信号。

    用法::

        class MyModel(ComponentModel):
            name: str = SignalProperty("")
            width: int = SignalProperty(100, emit='size_changed')

    属性变更时自动 emit 指定信号（默认 data_changed）。
    _loading=True 期间不 emit，用于 from_dict 批量加载。
    """
```

### 3.5 方法文档字符串

- 公有方法：应有文档字符串
- 私有方法（`_` 前缀）：可选，简单方法可省略
- 简单的 getter/setter：不需要文档字符串

## 4. 类型提示

### 4.1 使用 `typing` 模块（兼容性）

```python
# 正确 — 使用 typing 模块
from typing import Dict, Any, List, Optional, Tuple

def to_dict(self) -> Dict[str, Any]:
def _detect_squeeze(self) -> Tuple[bool, Tuple[int, int]]:

# 错误 — Python 3.9+ 内建泛型（项目需兼容更低版本）
def to_dict(self) -> dict[str, any]:
def _detect_squeeze(self) -> tuple[bool, tuple[int, int]]:
```

### 4.2 Optional 用于可能为 None 的参数

```python
# 正确
def lottery_animation(self, candidates: Optional[List[str]] = None,
                       duration_ms: int = 3000,
                       on_finished: Optional[Callable] = None) -> None:

# 错误
def lottery_animation(self, candidates: list = None, on_finished=None):
```

### 4.3 函数签名应有完整类型提示

```python
# 正确
def set_position(self, x: int, y: int) -> None:

# 可接受（简单方法）
def _sync_item_labels(self) -> None:
```

## 5. 核心设计模式

### 5.1 SignalProperty 描述符

这是项目最核心的模式，用于声明自动通知属性：

```python
class MyModel(ComponentModel):
    # 基本用法
    name: str = SignalProperty("")

    # 带验证
    opacity: float = SignalProperty(1.0, validator=lambda v: 0.0 <= v <= 1.0)

    # 带变换
    duration: int = SignalProperty(3000, transform=lambda v: max(500, min(30000, int(v))))

    # 自定义 emit 信号
    width: int = SignalProperty(120, emit=('size_changed', 'data_changed'),
                                 emit_args={'size_changed': lambda o: (o._width, o._height)})

    # 只读
    id: str = SignalProperty("", readonly=True)

    # 序列化别名
    type: str = SignalProperty("", readonly=True, serialize_as="comp_type")

    # 工厂默认值（可变类型）
    items: list = SignalProperty(default_factory=list)
```

**注意**：需要 `_sync_xxx` 的属性（如 items/item_labels）不能用 SignalProperty，
必须手写 `@property` + setter，因为赋值时需要调用同步方法。

### 5.2 Pydantic 数据模型

纯数据结构使用 Pydantic BaseModel，自动获得序列化和验证：

```python
class StyleConfig(BaseModel):
    background_color: str = "#f0f0f0"
    font_size: int = 12

# 使用
style = StyleConfig.model_validate(data)  # 从字典创建
data = style.model_dump()                  # 序列化为字典
```

### 5.3 枚举统一

所有枚举集中在 `models/schemas.py` 定义，继承 `str, Enum`：

```python
class ActionType(str, Enum):
    NONE = "none"
    CLOSE_PROGRAM = "close_program"
```

枚举值使用英文小写下划线，显示名称在 `DEFAULT_ACTIONS` 等处用中文。

### 5.4 组件注册表

新组件类型必须注册到 `ComponentRegistry`：

```python
# models/registry_init.py
ComponentRegistry.register('my_component', MyComponentModel)
```

### 5.5 单例模式

模块级单例通过全局变量 + getter 函数实现：

```python
_instance: Optional[MyManager] = None

def get_my_manager() -> MyManager:
    global _instance
    if _instance is None:
        _instance = MyManager()
    return _instance
```

### 5.6 EventBus 事件总线

组件间解耦通信使用 EventBus 发布-订阅模式：

```python
# 发布事件
from eventbus import event_bus
from eventbus.event_bus import ComponentMovedEvent
event_bus.publish(ComponentMovedEvent(comp_id="xxx", x=100, y=200))

# 订阅事件
event_bus.subscribe(ComponentMovedEvent, lambda e: print(e.comp_id, e.x, e.y))
```

所有事件类定义在 `eventbus/event_bus.py`，继承 `BaseEvent`。

### 5.7 Presenter 模式（MVP过渡）

Presenter 作为 Model 和 View 之间的桥梁，通过 EventBus 通信：

```python
class CanvasPresenter:
    def __init__(self):
        # 监听模型信号 → 发布 EventBus 事件
        model.position_changed.connect(self._on_position_changed)
        # 订阅 EventBus 事件 → 更新模型
        event_bus.subscribe(SomeEvent, self._handle_event)
```

Presenter 使用 `weakref` 避免循环引用，注意在析构时取消订阅。

## 6. 注释规范

### 6.1 注释语言：简体中文

### 6.2 有价值的注释

```python
# _loading=True 期间不 emit，用于 from_dict 批量加载
if not getattr(obj, '_loading', False):
    signal.emit()

# 修复：只关闭当前窗口，而不是所有窗口
if current_window:
    current_window.close()
```

### 6.3 避免冗余注释

```python
# 错误 — 注释只是重复代码
self._name = name  # 设置名称

# 正确 — 注释解释"为什么"
self._name = name or f"{comp_type}_{self._id}"  # 无名称时自动生成
```

### 6.4 信号入口注释（控制器专用）

`ProjectController` 中的信号连接使用统一格式的注释：

```python
# 【信号入口】主窗口项目操作信号 -> 控制器槽
self.window.new_project.connect(self._on_new_project)       # 新建项目请求 -> 创建新项目
self.window.save_project.connect(self._on_save_project)     # 保存项目请求 -> 保存项目
```

### 6.5 禁止调试输出

不要在代码中保留 `print()` 调试语句。使用 `debug_log()` 函数：

```python
# 正确
debug_log('position', f"组件移动: ({new_x}, {new_y})")

# 错误
print(f"组件移动: ({new_x}, {new_y})")
```

### 6.6 禁止裸 except

```python
# 正确
except (AttributeError, RuntimeError, TypeError):
    pass

# 错误
except:
    pass
```

## 7. 序列化约定

### 7.1 `comp_type` vs `type`

- Python 属性名：`type`（因为 `type` 是 Python 内建函数，但作为属性名可接受）
- 序列化键名：`comp_type`（通过 `serialize_as="comp_type"` 指定）
- 反序列化时同时检查两种键名：`data.get('comp_type') or data.get('type', '')`

### 7.2 to_dict / from_dict

- `ComponentModel` 基类自动处理所有 SignalProperty 字段
- 子类有手写 `@property` 属性时，需覆盖 `to_dict`/`from_dict` 处理额外字段
- Pydantic 模型使用 `model_dump()` / `model_validate()`

## 8. 已知问题与改进方向

### 8.1 重复代码待提取

| 重复项 | 出现位置 | 建议 |
|--------|---------|------|
| `TITLE_BAR_HEIGHT = 28` | canvas.py, runner.py, component_factory.py, container_renderer.py | 提取到 `styles/constants.py` |
| `debug_log()` 函数 | project_controller.py, runner.py | 提取到 `utils/debug.py` |
| `has_branch` / `action_params` | ButtonModel, ProgressBarModel | 提取到 `BranchMixin` |
| `_sync_item_labels` / `_sync_image_labels` | LotteryModel, ImageCarouselModel, AlternatingModel | 提取到 `SyncLabelsMixin` |
| 抽奖动画逻辑 | LabelModel, LotteryModel, ImageCarouselModel | 提取到 `LotteryAnimationMixin` |

### 8.2 构造函数风格不统一

- 大部分组件使用显式参数：`def __init__(self, name: str = "", ...)`
- `LotteryModel` / `ConfirmButtonModel` 使用 `**kwargs`

建议统一为显式参数风格，`**kwargs` 仅在子类需要透传基类参数时使用。

### 8.3 `exec()` 安全风险

- `action_executor.py` 中的 `exec(code)` 无任何限制
- `communication_system.py` 中的 `exec(code, {"__builtins__": {}})` 仍不安全
- 建议使用沙箱执行或完全禁用

### 8.4 硬编码样式字符串

视图中大量硬编码的 CSS 样式字符串，应统一使用 `styles/` 目录中的样式系统。
