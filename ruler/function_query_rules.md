# 函数查询工具使用规则

## 1. 基本使用原则

### 1.1 查询范围
- 优先查询与当前任务直接相关的函数
- 使用关键词搜索功能快速定位目标函数
- 注意函数的完整路径（包括类名）以避免命名冲突

### 1.2 结果解读
- 调用关系表示函数直接或间接调用了其他函数
- 被调用关系表示其他函数直接或间接调用了该函数
- 高调用数量通常表示函数是核心工具函数或框架函数
- 高被调用数量通常表示函数是重要的业务逻辑入口或事件处理函数

## 2. 性能优化规则

### 2.1 缓存利用
- 重复查询同一函数时会自动使用缓存结果
- 首次查询后建议保存查询对象以复用缓存
- 缓存TTL为5分钟，过期后会自动重新查询

### 2.2 查询策略
- 先使用搜索功能定位目标函数，再获取详细信息
- 使用快照功能了解整体项目结构后再深入具体函数
- 对于复杂的调用链分析，建议使用get_call_chain方法

## 3. 调用链分析规则

### 3.1 深度控制
- 建议调用链深度不超过5层，避免信息过载
- 根据实际需求调整max_depth参数
- 注意循环调用会被自动过滤

### 3.2 路径解读
- 调用链按执行顺序排列
- 每个节点表示实际会被执行的函数
- 路径中的函数可能来自不同模块或类

## 4. 常见使用场景

### 4.1 函数影响分析
- 当修改一个函数时，使用get_called_functions查看可能受影响的函数
- 使用get_caller_functions查看可能需要重新测试的调用者
- 结合调用链分析了解传播影响范围

### 4.2 代码重构参考
- 高被调用函数需要谨慎修改，考虑向后兼容性
- 高调用但低被调用的函数通常是安全的工具函数
- 查看函数定义了解其实现复杂度和副作用

### 4.3 性能优化
- 关注高调用函数的性能，优化收益最大
- 检查是否存在不必要的重复调用
- 分析调用链中是否存在性能瓶颈

## 5. 注意事项

### 5.1 数据准确性
- 分析结果基于静态代码分析，动态调用可能不完整
- 某些高级特性（如反射、动态加载）的调用可能未被捕获
- 建议结合运行时分析工具进行验证

### 5.2 命名约束
- 函数名包含完整的类路径（如 ClassName.method_name）
- 使用简单名时可能匹配多个函数，请使用完整名以确保准确性
- 构造函数、析构函数和特殊方法均包含在分析中

### 5.3 性能考虑
- 首次加载分析数据可能需要较长时间（取决于项目大小）
- 后续查询将受益于缓存机制
- 极大项目建议增加缓存TTL或持久化缓存到磁盘

## 6. 最佳实践示例

### 6.1 新功能集成前
```
# 检查目标函数的现有调用情况
info = query.get_function_info("target_function")
print(f"目标函数被 {len(info['called_by'])} 个函数调用")
print(f"目标函数调用 {len(info['calls'])} 个其他函数")
```

### 6.2 错误排查时
```
# 查看异常函数的调用者和被调用函数
info = query.get_function_info("error_function")
callers = info['called_by']
calls = info['calls']
# 重点检查调用者的输入参数和被调用函数的返回值
```

### 6.3 代码审查时
```
# 检查新函数的设计合理性
info = query.get_function_info("new_function")
if len(info['calls']) > 20:
    print("警告: 函数调用过多，考虑解耦")
if len(info['called_by']) == 0 and not info['name'].startswith("_"):
    print("提示: 公共函数未被其他函数调用，检查是否需要导出")
```

## 8. 项目文档参考

### 8.1 项目概述
UI快速开发工具是一个基于 PyQt6 的可视化 UI 设计工具，允许用户通过拖拽方式创建桌面应用程序界面。支持 Galgame 模式的多窗口/事件管理。

### 8.2 核心功能
- **可视化设计**：拖拽式组件设计，所见即所得
- **组件系统**：支持按钮、标签、输入框、容器等多种组件
- **属性编辑**：完整的属性面板，实时预览修改效果
- **行为配置**：为组件配置点击行为（打开文件、运行命令、显示消息等）
- **项目保存/加载**：JSON格式项目文件，方便分享和版本控制
- **实时运行**：一键运行设计的应用程序

### 8.3 项目结构
项目采用分层架构：
- **main.py**：程序入口，AppManager 管理欢迎页和编辑器切换
- **models/**：数据模型层，包含基础模型和具体组件模型
- **views/**：视图层，包括主窗口、设计画布、组件树、属性面板等
- **controllers/**：控制器层，协调模型和视图
- **runtime/**：运行时模块，包括行为执行器和运行时启动器
- **utils/**：工具类，包括设置、撤销管理器、字体工具、崩溃日志等

### 8.4 关键模块说明
- **AppInitializer**：负责应用初始化，已将原 AppManager 的初始化职责分离
- **AutoSaveService**：负责自动保存逻辑
- **TempFileManager**：负责临时文件管理
- **SessionLogger**：会话日志记录器
- **RendererFactoryV2**：渲染器工厂，负责预加载核心渲染器
- **LayoutRefreshManager**：布局刷新管理器，解决文本居中问题
- **LeftPanelManager**：左侧面板管理器，解决逻辑树显示问题
- **ViewManager**：视图管理器，负责视图切换

### 8.5 使用方法
1. **启动程序**：`python main.py`
2. **基本操作**：
   - 添加组件：点击工具栏的"+按钮"、"+标签"等按钮
   - 移动组件：在画布上拖拽组件
   - 调整大小：选中组件后拖拽边缘手柄
   - 编辑属性：在右侧属性面板修改组件属性
   - 配置行为：点击属性面板的"配置行为"按钮
   - 保存项目：文件 → 保存（或Ctrl+S）
   - 运行项目：按F5或点击运行按钮
3. **常用快捷键**：
   - Ctrl+N：新建项目
   - Ctrl+O：打开项目
   - Ctrl+S：保存项目
   - Ctrl+Z：撤销
   - Ctrl+Y：重做
   - F5：运行项目
   - Delete：删除选中组件

### 8.6 技术栈
- Python 3.8+
- PyQt6：GUI框架
- MVC架构：模型-视图-控制器分离
- Signal-Slot机制：组件间通信
